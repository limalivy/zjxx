# 字界修仙 — 设计规格

## 概述

"字界修仙"是一款基于浏览器的中文打字游戏。玩家通过输入屏幕上下落的中文字符来获取分数，错误输入或字符触底会扣除血量，血量归零则游戏结束。游戏以修仙为主题视觉风格。

## 技术选型

- 纯 HTML + JS + CSS，不依赖任何前端框架
- 三个文件：`index.html`、`style.css`、`game.js`
- 游戏循环由 `requestAnimationFrame` 驱动（60fps）


```js
charObj = {

  speed: 0.8,            // 下落速度（px/帧）
```

## 游戏机制

### 游戏循环（game.js）

每帧执行：
1. 所有活跃字的 `y += speed`
2. 更新 DOM 位置
3. 检测越界：`y > window.innerHeight` → 扣 2 HP，移除该字
4. 检测 HP ≤ 0 → 触发游戏结束
5. 渲染下一帧

### 字符生成

- 从 200 个常用汉字组成的字库中随机抽取
- 所有字从屏幕顶部上方生成（y = 随机 -30 ~ -120），水平位置 X 随机（留出左右边距）
- 速度随机（0.5 ~ 1.2 px/帧），不同速度自然形成垂直方向均匀散开
- 定时刷新 + 打掉即补共同保证屏幕上的字在 5 个以内且节奏紧凑
- 最多同时存在 5 个字

### 刷新机制（混合模式）

- **定时刷新**：每 2.5 秒自动生成 1 个新字（当前屏幕 < 5 个字时）
- **打掉即补**：正确输入消除 1 个字后，立即补 1 个新字
- 始终保持屏幕 ≤ 5 个字

### 输入匹配

- 监听 `compositionend` 事件获取最终输入的中文字符
- 也用 `input` 事件兜底处理无需 IME 的输入（如英文/数字键盘直接输入）
- 匹配判定逻辑：
  1. `compositionstart` → 标记 `isComposing = true`，跳过匹配
  2. `compositionend` → 标记 `isComposing = false`，取 `e.data` 作为输入字符
  3. `input` 事件 → 若 `!isComposing`，取输入框内容作为输入字符
- 每次获得输入字符后，遍历 `activeChars` 做精确匹配（`===`）
- 匹配成功 → +10 分，消除该字（CSS 消散动画），立即补新字
- 匹配失败 → -1 HP，触发全屏红色闪烁
- 每次判定后清空输入框

### 扣血机制

| 事件 | 扣血量 |
|------|--------|
| 字掉出屏幕底部 | -2 HP |
| 输入不匹配（打错字） | -1 HP |

### 全屏闪烁反馈

扣血时触发：
- 全屏覆盖一层红色半透明遮罩
- CSS transition：0ms 出现 → 300ms 淡出
- 不阻塞游戏循环

### 游戏结束

HP ≤ 0 时：
- `isGameOver = true`
- 停止游戏循环和定时器
- 清除屏幕所有掉落字
- 显示结束遮罩层，展示最终得分
- 提供"重新开始"按钮

### 视觉风格

- **背景**：深色（#1a1a2e 或类似深蓝黑）
- **文字**：金色（#d4a574 / #ffd700）
- **字体**：优先使用楷体（KaiTi / STKaiti），回退到系统 serif
- **掉落字**：较大字号（2.5rem+），白色/金色
- **HP 血条**：渐变（绿 → 黄 → 红），随血量变化
- **全屏闪烁**：红色半透明遮罩，0→300ms 淡出

## 状态转换

```
开始（页面加载）→ 游戏中 → HP ≤ 0 → 游戏结束 → 重新开始 → 游戏中
```

无开始页面和菜单，打开即玩。游戏结束后显示遮罩层覆盖在游戏画面上。

## 中文输入法兼容

使用 `compositionstart` 和 `compositionend` 事件：
- `compositionstart` 时设置 `isComposing = true`
- `compositionend` 时设置 `isComposing = false`，用最终输入值触发匹配判定
- `input` 事件中，如果 `isComposing` 则跳过匹配（避免拼音中间状态触发判定）
